# 代码变更分析任务

## 输入数据
以下是 Git Commit 的 diff 内容：
```
{diff_content}
```

## 分析要求
请作为资深代码审查专家，分析此代码变更的核心技术逻辑。

### 重点分析维度
- **业务逻辑变更**：新增、修改或删除的核心功能逻辑
- **架构调整**：模块间关系、接口定义、数据流变化
- **算法优化**：性能改进、计算逻辑调整
- **错误处理**：异常处理机制、边界条件处理
- **数据模型**：数据结构、数据库 schema 变更

### 忽略内容
- 代码格式化、空格调整、注释修改
- 依赖文件更新（package-lock.json, *.lock 等）
- 配置文件中的非功能性调整
- 重命名操作（除非伴随逻辑变更）

## 输出规范
- **格式**：单句总结，不超过50字
- **内容**：聚焦技术实质，避免描述性语言
- **粒度**：达到代码审查级别精度

## 示例参考
✅ 正确："重构用户认证模块，引入JWT令牌替换Session机制"
✅ 正确："优化数据库查询，添加复合索引提升列表页性能"
❌ 错误："修改了一些代码"（过于模糊）
❌ 错误："修复bug"（缺乏技术细节）
